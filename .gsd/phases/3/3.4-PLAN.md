# Plan 3.4: Content Peeking

## Objective
Implement a multi-strategy file peeking dispatcher to extract text content from PDFs and other types for content-aware routing.

## Context
- docs/SPEC.md: Tier 3 Peek methods
- docs/ARCHITECTURE.md: internal/peek/
- docs/TASKS.md: Task 3.4

## Tasks

<task type="auto">
  <name>Text and Docx Peeking Strategies</name>
  <files>internal/peek/text.go</files>
  <action>
    Implement `TextPeek(path string)` and `DocxPeek(path string)` (using optional pandoc).
    - Read first 2000 bytes for plain text/md.
    - Gracefully substitute invalid UTF-8 bytes.
    - If `pandoc` is available, extract raw text from `.docx`.
  </action>
  <verify>go test ./internal/peek/...</verify>
  <done>Text content is correctly extracted from plain text and markdown files.</done>
</task>

<task type="auto">
  <name>PDF Peeking and Dispatcher</name>
  <files>internal/peek/pdf.go, internal/peek/peek.go</files>
  <action>
    Implement `PdfPeek(path string)` and a global dispatcher in internal/peek/peek.go.
    - `PdfPeek`: Call `pdftotext -l 1 -` to get text from the first page (max 800 chars).
    - `PeekDispatcher`: Route based on extension; return "" if unable to peek.
  </action>
  <verify>go test ./internal/peek/...</verify>
  <done>PDF content corresponds to the first page text; dispatcher routes all supported formats.</done>
</task>

## Success Criteria
- [ ] First page text is successfully extracted from PDFs.
- [ ] Text content is accurately peeked for .txt, .md, .rst files.
- [ ] Dispatcher correctly routes different extensions.
