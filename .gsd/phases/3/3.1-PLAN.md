# Plan 3.1: Initialization Command

## Objective
Implement `sortd init` to provide a seamless first-run experience. This command will scaffold the configuration, prepare the database, perform the initial folder indexing, and install the systemd user service.

## Context
- cmd/sortd/main.go
- internal/config/config.go
- sortd.service

## Tasks

<task type="auto">
  <name>Implement sortd init</name>
  <files>cmd/sortd/main.go</files>
  <action>
    - Define `initCmd`.
    - Logic:
      1. Call `config.LoadConfig` which handles `config.toml` creation if missing.
      2. Ensure `.local/share/sortd` is created (for DB/Logs).
      3. Reuse the `index` logic to perform a crawl of standard folders (Documents, Desktop, Downloads).
      4. If a `sortd.service` file exists in the current directory, copy it to `~/.config/systemd/user/sortd.service`.
      5. Inform the user to run `systemctl --user daemon-reload` and `systemctl --user enable --now sortd`.
    - Register `initCmd` to `rootCmd`.
  </action>
  <verify>go build ./cmd/sortd/... && ./sortd init --help</verify>
  <done>sortd init command is implemented and discoverable.</done>
</task>

## Success Criteria
- [ ] `sortd init` scaffolds all necessary folders and files.
- [ ] `sortd init` performs a successful initial index.
- [ ] Systemd service is placed in the correct user directory.
