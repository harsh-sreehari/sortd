# Plan 5.1: NLP Review Routing

## Objective
Enable a conversational approach to clearing the `.unsorted` backlog by allowing users to describe what a file is rather than typing a full path.

## Context
- internal/pipeline/tier2.go
- internal/llm/backend.go
- cmd/sortd/main.go

## Tasks

<task type="auto">
  <name>Implement Description-to-Folder Matching (Fuzzy)</name>
  <files>internal/pipeline/tier2.go</files>
  <action>
    - Add `MatchDescription(desc string, folders []graph.FolderIndex) (string, bool)` to `tier2.go`.
    - Tokenize the user description and find the best overlapping folder from the index.
    - Set a high threshold (e.g. 0.5) to avoid false positives.
  </action>
  <verify>grep "func MatchDescription" internal/pipeline/tier2.go</verify>
  <done>Fuzzy folder matching from natural language descriptions is available.</done>
</task>

<task type="auto">
  <name>Implement LLM Review Resolution</name>
  <files>internal/llm/backend.go</files>
  <action>
    - Add `ResolveReview(userInput string, filename string, tree []string) (TagResponse, error)` to `LLMBackend`.
    - Update `LMStudioBackend` to implement this.
    - Prompt: "User says this file ('%s') is '%s'. Based on these folders %v, where should it go?"
  </action>
  <verify>grep "ResolveReview" internal/llm/backend.go</verify>
  <done>LLM can now interpret casual user input for file routing.</done>
</task>

<task type="auto">
  <name>Integrate NLP into Review Command</name>
  <files>cmd/sortd/main.go</files>
  <action>
    - Update `reviewCmd` to fetch `store.UnsortedFiles()`.
    - For each file, take user input.
    - Routing Logic:
        1. If input is an absolute/relative path that exists: Move directly.
        2. If `MatchDescription` finds a strong match: Move there.
        3. Fallback: Call `ResolveReview` on the LLM.
    - After move, call `store.MarkCorrected(logID, finalPath, matchedFolder)` to update affinities.
  </action>
  <verify>go build ./cmd/sortd/... && ./sortd review --help</verify>
  <done>Review command is now conversational and updates system affinities.</done>
</task>

## Success Criteria
- [ ] User can type "taxes for 2024" and have a file routed to "Documents/Tax/2024" (if it matches index).
- [ ] LLM fallback handles ambiguous descriptions.
- [ ] Corrected files update the affinities table for future 100% automated sorting.
