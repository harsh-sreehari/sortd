# Plan 4.1: Enhanced Logging & Search

## Objective
Overhaul the `sortd log` experience with rich terminal formatting (colors, spacing), filtering capabilities, and a dedicated `find` command for searching history.

## Context
- internal/store/store.go
- cmd/sortd/main.go

## Tasks

<task type="auto">
  <name>Implement Advanced Search Store Helper</name>
  <files>internal/store/store.go</files>
  <action>
    - Add `SearchLog(n int, filter map[string]string) ([]LogEntry, error)` to the Store.
    - Support filters: `tag`, `tier`, `action`, `query` (LIKE match on filename/reasoning), and `today` (timestamp filter).
    - Ensure it returns `LogEntry` slice sorted by newest first.
  </action>
  <verify>grep "func (s *Store) SearchLog" internal/store/store.go</verify>
  <done>Store can perform complex filtered queries on history.</done>
</task>

<task type="auto">
  <name>Overhaul Log Command</name>
  <files>cmd/sortd/main.go</files>
  <action>
    - Update `logCmd` to use `SearchLog`.
    - Add flags: `--tier`, `--action`, `--parked`, `--today`, `--tag`.
    - Implement ANSI color coding:
        - "moved": Green
        - "parked": Yellow
        - "skipped": Gray
        - Tier headers: Cyan
    - Improve table layout with better padding and tag display.
  </action>
  <verify>go build ./cmd/sortd/... && ./sortd log --help</verify>
  <done>Log command is colorful, structured, and filterable.</done>
</task>

  <done>Find command is implemented.</done>
