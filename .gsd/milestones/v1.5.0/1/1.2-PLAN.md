Implement placeholder helper methods in the `store` package so that the future CLI commands can process parked files and learn from human corrections using the `affinities` table.

  <action>
    - Update `MarkCorrected(id int, newDest string, folderMatch string) error` (modify signature if necessary). 
    - The logic MUST: 
       1. Execute an `UPDATE sort_log SET corrected = 1, destination = ? WHERE id = ?`.
       2. Fetch the `tags` for that `id` from the log table (and unmarshal the tags JSON back to a string slice).
       3. For every tag in the slice, insert into the `affinities` table using an upsert to increment the weight: e.g. `INSERT INTO affinities (tag, folder, weight) VALUES (?, ?, 1.0) ON CONFLICT(tag, folder) DO UPDATE SET weight = weight + 1.0`. Where folder is the semantic folder matching directory.
  </action>

  <done>MarkCorrected updates the log and positively reinforces the affinities table</done>
</task>
