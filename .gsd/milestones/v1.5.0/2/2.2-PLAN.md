# Plan 2.2: Daemon Management & PID Files

## Objective
Implement proper lifetime management for the daemon. This includes writing a PID file on start, and using it for `stop` and `status` commands. Also, tune the systemd service dependency.

## Context
- cmd/sortd/main.go
- sortd.service

## Tasks

<task type="auto">
  <name>PID File Logic</name>
  <files>cmd/sortd/main.go</files>
  <action>
    - Define a standard PID path: `~/.local/share/sortd/sortd.pid`.
    - In `daemonStartCmd`:
        - Check if PID exists and process is running. If so, abort.
        - Write current PID to file.
        - Use a `defer` to remove the PID file on shutdown.
    - Implement `daemonStopCmd`:
        - Read PID, send `SIGTERM`.
        - Wait briefly for exit.
    - Implement `daemonStatusCmd`:
        - Check PID file. Verify if process is active.
  </action>
  <verify>go build ./cmd/sortd/...</verify>
  <done>Daemon now manages a PID file and has stop/status commands.</done>
</task>

<task type="auto">
  <name>Systemd Dependency Tuning</name>
  <files>sortd.service</files>
  <action>
    - Update `After=` to include `graphical-session.target` (since LM Studio is often a GUI app).
    - Ensure it's robust for a local user-session.
  </action>
  <verify>grep "After=" sortd.service</verify>
  <done>Systemd unit has optimized dependencies.</done>
</task>

## Success Criteria
- [ ] `sortd daemon stop` correctly kills the watcher.
- [ ] PID file prevents double-starts.
- [ ] Systemd unit is updated.
