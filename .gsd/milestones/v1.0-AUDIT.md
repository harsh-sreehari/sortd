## Concerns
- Potential for high local CPU/resource footprint during LLM invocation since it triggers sequentially synchronously to folder watch events.
- No `review` interactive logic was solidly completed entirely (the template logic states "interactive review logic initialized (mock)"). This means users can't legitimately flush the `.unsorted/` directory cleanly directly from the prompt yet.
- Same with `log` history command, the DB was integrated but the `logCmd` only states "mock listing logic". 

## Recommendations
1. Fully implement the SQL read operations for `logCmd`.
2. Map standard Go `bufio.Scanner` prompt logic to actually clear and sort `.unsorted` for `reviewCmd`. 
3. Re-structure LLM tasks asynchronously out of the watcher if load slows down event cycles.
